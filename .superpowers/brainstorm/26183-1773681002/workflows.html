Key Design Decisions

New: Workflow E closes the loop

The missing feedback cycle. After launch, analyze A/B tests and funnels, synthesize learnings, re-prioritize the backlog, and feed insights back into discovery or the next sprint. This turns the linear pipeline into a continuous cycle.

New: Orphan skills integrated into workflows

launch-checklist-generator now caps Workflow C. sales-enablement-kit joins Workflow D. stakeholder-simulator, meeting-notes-processor, and executive-update-generator remain standalone utilities.

Workflows are guides, not cages

Any skill can be run standalone via /manna [skill-name]. Workflows just provide sequencing, dependency checks, and state tracking. The PM can skip steps, jump in mid-workflow, or run skills ad-hoc.
