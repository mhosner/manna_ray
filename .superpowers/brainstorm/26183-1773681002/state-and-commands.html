Command Interface
PROJECT SETUP
  /manna init                        Scaffold project (context/, outputs/, state)
  /manna context init                Create context file templates
  /manna context check               Show context file health (current/stale/missing)
  /manna context update [file]       Guided update of a context file

SKILL EXECUTION
  /manna [skill-name]                Run a single skill (e.g., /manna prd-generator)
  /manna [skill-name] --skip-checks  Run without dependency/staleness checks
  /manna list                        List all available skills by mode
  /manna list discovery              List skills in a specific mode

WORKFLOW MANAGEMENT
  /manna workflow start [name]       Begin a workflow (e.g., idea-to-sprint)
  /manna workflow status             Show active workflow progress
  /manna workflow next               Advance to next step in active workflow
  /manna workflow skip               Skip current step, advance to next
  /manna workflow list               List available workflows

STATUS & HISTORY
  /manna status                      Dashboard: context health + active workflows + recent runs
  /manna history                     Full run history with output links
  /manna outputs [skill]             List outputs from a specific skill

Interaction Model

Workflow: guided but not rigid

/manna workflow start idea-to-sprint → runs dependency checks → starts at step 1 (prioritization-engine). After each skill completes, the PM can:

  • /manna workflow next — advance to next step (auto-loads previous output as input)
  • /manna workflow skip — skip this step
  • Do something else entirely, then come back with /manna workflow next later

Workflows persist across Claude Code sessions via state.json.

Ad-hoc: no workflow required

/manna stakeholder-simulator just runs the skill directly. Context is still loaded and validated, but there's no workflow tracking. The run still appears in history.

Status: at-a-glance awareness

/manna status shows a unified dashboard:

  • Context health (3 current, 1 stale, 1 missing)
  • Active workflow (idea-to-sprint: step 3/6)
  • Last 5 skill runs with output links
